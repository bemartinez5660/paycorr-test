Características implementadas (Descripción para facilitar la prueba de todas las características)

  • Implementación de enrutamiento con al menos una carga con Lazy Loading con motivo ilustrativo.

  • Modelado de la entidad de la respuesta del api del NY Times

  • Creación de componentes modulares y reutilizables.

  • Manejo de excepciones y se muestra una imagen de placeholder con el logo de 2innovate en caso de error al cargar la imagen ya sea pq se esta offline o porque no viene en la respuesta del api en el arreglo de multimedias que algunos vienen vacíos

  • Implementación de paginación

  • Indicador de carga al cargar los artículos, en todos los eventos de carga.

  • Interceptor global para agregarle los QueryParams fijos que siempre se van a enviar como, el "api-key", el "begin_date" y el "end_date"

  • Consulta a la API de NY Times por las noticias del día actual.

  • Al obtener las noticias de forma correcta las mismas se presentan en la tabla (ocultando el indicador de progreso). En caso de ocurrir un error se presentará un mensaje acorde al usuario.

  • Una vez obtenidas las noticias estas deben estar disponibles cuando el usuario no cuente con conexión (deben ser persistidas localmente).

  • Persistencia de los articulos en el localStorage del navegador, manejando correctamente al almacenarlos evitando la duplicidad de los elementos

  • Obtención de las noticias de forma manual, mediante el botón de refresh a la derecha en la barra de navegación, usando el mismo proceso de mostrar el indicador de carga cuando se cargan los siguientes elementos

Características adicionales fuera de los requisitos solicitados:

  • Integración con Tailwind CSS framework que facilita agregar estilos.
